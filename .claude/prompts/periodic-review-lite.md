PERIODIC REVIEW: Pause and regroup before continuing.

## Architecture Review
- Restate the original task. What were you asked to do?
- Based on everything you've learned during implementation, is your current approach still correct? Are there problems, edge cases, or constraints you've discovered that challenge it?
- If what you've learned invalidates the current approach, STOP implementing. Reason about the correct architecture based on what you now know. If a task file exists, update it with the new architecture and the reasoning behind the change. Revert changes that conflict with the new approach (keep changes that are still valid). Then resume with the corrected approach.

## Progress and Current Work
- What have you accomplished so far? What's left?
- What are you working on right now? How does it connect to the original task?

## Scope Check
- Are you only making changes required for the task? If you've touched files, added code, or made improvements that aren't directly needed, revert them.

## Stuck Check
- Are you facing a problem? If yes, state it clearly.
- Do you understand the root cause? If not, stop and research before continuing to push fixes.
- If you've been trying the same approach and it isn't working, zoom out. Reconsider the problem from a higher level before continuing.

After completing this review, continue working.
